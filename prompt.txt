1. Installing Claude Code

1.1 Install on Your Personal PC (RTX 3060)
Open a terminal (PowerShell on Windows, Terminal on Mac/Linux) and run these commands one at a time:

# Step 1: Install Node.js first (Claude Code requires it)
# Go to https://nodejs.org and download the LTS version
# Or on Linux/Mac:
curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
sudo apt-get install -y nodejs

# Step 2: Install Claude Code globally
npm install -g @anthropic-ai/claude-code

# Step 3: Verify installation
claude --version

# Step 4: Login (you need an Anthropic account with Claude Code access)
claude login

1.2 Install on University PC #53
Do the exact same steps via your remote desktop or SSH connection to the university PC. Make sure you log in to the same Anthropic account on both machines.

IMPORTANT: Claude Code requires an internet connection to work. Make sure both machines can reach the internet. Your university PC needs it whenever you use Claude Code, even if your training code runs offline.

2.2 Write Your CLAUDE.md File
This is the most important setup step. Claude Code reads CLAUDE.md automatically every session. It is your persistent memory. Open it in any text editor and paste this:

# CLAUDE.md — Project Context for Claude Code

## Project Name
Spectral-KAN: Interpretable Masked Autoencoding for Biosignals via KAN Decoder

## What We Are Building
A Masked Autoencoder (MAE) where the decoder is a Kolmogorov-Arnold Network (KAN).
We pretrain on biosignals (ECG from PTB-XL, audio from ESC-50).
After training, we analyze the learned KAN spline edges for interpretability.

## Architecture
- Encoder: ViT-Small (12 layers, 384 hidden dim, 6 heads)
- Decoder (baseline): Transformer decoder, 2 blocks
- Decoder (ours): KAN-based decoder using efficient-kan library
- Masking: 75% of patches masked during pretraining

## Hardware
- Personal PC: RTX 3060 12GB, use for small experiments
- University PC: RTX 4070 Ti Super 16GB, use for full runs

## Key Libraries
- PyTorch 2.x with CUDA
- timm (for ViT encoder)
- einops (for tensor reshaping)
- efficient-kan (for KAN layers)
- torchaudio (for audio processing)
- wfdb (for ECG/PTB-XL loading)
- scipy, matplotlib, wandb

## Datasets
- ESC-50: /data/ESC-50/ (2000 audio clips, 50 classes)
- PTB-XL: /data/ptbxl/ (21837 ECG records, 12 leads)

## Key File Locations
- models/encoder.py: ViT encoder
- models/decoder_transformer.py: baseline decoder
- models/decoder_kan.py: KAN decoder (our contribution)
- models/mae.py: full MAE framework
- data/ecg_dataset.py: PTB-XL loader
- data/audio_dataset.py: ESC-50 loader
- scripts/train.py: main training script
- scripts/evaluate.py: linear probe evaluation
- scripts/freq_band_analysis.py: frequency-band reconstruction analysis
- utils/edge_tracker.py: KAN edge statistics and snapshot helpers

## DO NOT
- Do not use ViT-Large (too big for our GPU)
- Do not use batch size > 128 on RTX 3060
- Do not use batch size > 256 on RTX 4070 Ti Super
- Do not change the masking ratio from 0.75 without updating the config

WHY THIS MATTERS: Every time you start Claude Code, it reads this file. This means Claude Code always knows your architecture, your datasets, your hardware limits, and your file structure — without you having to re-explain everything each session.

4. Phase 2: Building the Models (Week 2-4)

4.1 Build the ViT Encoder

PROMPT TO TYPE INTO CLAUDE CODE:
Create models/encoder.py with a SpectralViTEncoder class.
It should:
- Accept input spectrograms of any shape
- Divide the spectrogram into non-overlapping patches
- For ESC-50: input (1, 128, T), patch_size=16
- For PTB-XL: input (12, F, T), patch_size=16
- Use timm to build a ViT-Small backbone
  (embed_dim=384, depth=12, num_heads=6)
- Implement random masking: randomly mask 75% of patches
  and return only the visible 25%
- Also return the mask indices so the decoder knows
  which patches to reconstruct
- Include a method patch_to_img() to convert
  patch predictions back to image/spectrogram format
After writing, test it: pass a fake batch of shape
(4, 1, 128, 128) through the encoder and print
the output shape and mask shape.

4.2 Build the Transformer Decoder (Baseline)

PROMPT TO TYPE INTO CLAUDE CODE:
Create models/decoder_transformer.py with a
TransformerDecoder class. This is our baseline decoder.
It should:
- Take the encoded visible patches and the mask tokens
- Reconstruct the full sequence (visible + masked positions)
- Use 2 standard Transformer blocks
  (embed_dim=512, num_heads=8, mlp_ratio=4.0)
- Output a patch prediction for every masked position
- Match the reconstruction to the original patch values
Include a count_parameters() method that prints
the total number of trainable parameters.
Test by passing a fake encoder output through it.

4.3 Build the KAN Decoder (Your Core Contribution)
This is the most important file in your entire project. Take care with this prompt.

PROMPT TO TYPE INTO CLAUDE CODE:
Create models/decoder_kan.py with a KANDecoder class.
This is our main contribution — a KAN-based decoder for MAE.

Architecture design:
- Input: encoded visible patches + mask tokens
  (same interface as TransformerDecoder for easy swapping)
- Option A (default): A hybrid design with:
  1. One standard multi-head self-attention layer
     (to handle positional relationships between patches)
  2. Then two KAN layers using efficient-kan's KANLinear:
     (input_dim -> hidden_dim -> output_dim)
     where hidden_dim = 512 and output_dim = patch_size^2 * channels
  Use KANLinear with grid_size=5 and spline_order=3
- Output: patch predictions for masked positions

Also implement:
- get_edge_functions(num_points=200): extracts all KAN
  spline edge functions as numpy arrays for interpretability analysis
  Returns dict: {layer_name: array of shape (in_dim, out_dim, num_points)}
- count_parameters(): total trainable params

Use efficient-kan library (from the efficient_kan package).
Test it: pass same fake input as TransformerDecoder,
confirm output shapes match. Print parameter count.
Fix any import errors.

4.4 Build the Full MAE Framework

PROMPT TO TYPE INTO CLAUDE CODE:
Create models/mae.py with a MaskedAutoencoder class.

  3. Returns MSE for each band separately
  This is needed for our Claim 1 evaluation.

Create a factory function build_mae(config) that reads
a config dict and builds the full MAE with the right decoder.

and running a forward pass with fake data.

7. Phase 5: The Interpretability Analysis (Week 15-18)

7.1 Build the Edge Extraction and Visualization Tool

PROMPT TO TYPE INTO CLAUDE CODE:
Use utils/edge_tracker.py as the core KAN edge analysis helper.
It should:
1. Load a trained KAN decoder checkpoint
2. Call decoder.get_edge_functions(num_points=500) to extract
   all spline edge functions as smooth curves
3. For each edge function array (shape: in_dim x out_dim x 500):
   a. Compute total variation (how 'wiggly' the curve is):
      TV = sum of |f(x_i+1) - f(x_i)|
   b. Compute peak frequency using FFT on the curve values
   c. Compute curvature using second derivative
4. Sort all edges by total variation (most active = highest TV)
5. Create figure 1: grid plot of top 36 most active edges
   (6x6 grid, each subplot shows one edge function curve)
   Save to results/kan_edges_top36.png
6. Create figure 2: histogram of peak frequencies across all edges
   Save to results/kan_edges_freq_distribution.png
7. Save all edge statistics to results/edge_statistics.csv
   with columns: layer, in_dim, out_dim, total_variation,
   peak_frequency, curvature
Run it on the PTB-XL KAN checkpoint and show me the results.

7.3 Symbolic Regression on Active Edges

PROMPT TO TYPE INTO CLAUDE CODE:
Create scripts/symbolic_regression.py to try to find
mathematical formulas for the most interpretable edges.

For each of the top 20 edges by total variation:
1. Extract the edge curve as (x_values, y_values) pairs
2. Try fitting these candidate functions using scipy curve_fit:
   - sin(ax + b)
   - Gaussian: a * exp(-b * (x-c)^2)
   - Sigmoid: a / (1 + exp(-b*(x-c)))
   - Polynomial: ax^2 + bx + c
   - Absolute value: a * |x - b| + c
3. For each candidate, compute R-squared goodness of fit
4. Pick the best fitting function (highest R^2) if R^2 > 0.85
5. Create figure 5: for each edge with a good symbolic match,
   plot the raw spline curve and the fitted symbolic function
   side by side, with the formula and R^2 shown
   Save to results/symbolic_matches.png
6. Save all symbolic regression results to
   results/symbolic_results.csv with columns:
   edge_id, best_function, R_squared, parameters

11. Managing Long Sessions and Context

11.1 The Problem with Long Sessions
Claude Code has a context window limit. In a long session (2+ hours), it may start to forget earlier parts of the conversation or make mistakes because its context is full. Signs of this: it starts ignoring your CLAUDE.md, it forgets file locations, or it repeats code that already exists.

11.2 How to Reset Without Losing Progress
When you notice context drift, do this:

# Inside Claude Code, type:
/compact

# Or start a fresh session:
/exit
# Then reopen:
claude

# And use your session start ritual prompt from Section 3.1
